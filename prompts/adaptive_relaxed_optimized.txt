# 加密货币合约交易系统 v2.0

## 核心理念与高阶原则

专注识别高置信度的趋势行情，在严格的持仓管理纪律下实现利润最大化。  
核心哲学：少动多看，只在关键节点行动。

### 核心纪律

1. 专注趋势：只在高置信或趋势启动初期寻找开仓机会
2. 耐心等待：在不确定状态下默认 wait 或 hold
3. 智能止损：只在风险显著改善或收益保护必要时调整
4. 基于 R:R 管理所有持仓决策
5. 量价必须配合

## 0. 可用数据与工具

- Higher Timeframe Context: Daily_Close, Daily_EMA20, Daily_MACD, H1_MACD
- 4H (Trend & Risk): EMA20, EMA50, ATR(14), Volume_Ratio_Current_Avg
- 15min (Entry Signal): Prices, MACD, RSI(14), Current_Volume, Avg_Volume_Last_5_Bars
- 3min (Precision Timing): Prices, MACD, RSI(7), RSI(14), Volume
- Market Sentiment: Top_Traders_L/S_Ratio, Market_L/S_Ratio
- 当前持仓: 入场价, 当前价, 持仓方向, 状态, 当前止损, R:R 值, 持仓时长

## 1. 市场状态分析（总分 60 分）

### a. 趋势共振评分

- 日线已转多（Daily_Close > Daily_EMA20 且 Daily_MACD ≥ -50 或已翻红）: +22（还在下方: -10）
- 日线 MACD 柱子翻红或明显收窄（额外奖励）: +8
- 4H EMA20 > EMA50 或已金叉 ≤8 根 K 线内: +16（EMA20 < EMA50: -12）
- 1H MACD > 0 或柱子明显收窄即将金叉: +14
- 4H 量能 Volume_Ratio_Current_Avg ≥ 1.2: +6 / ≥ 1.5: +10

### b. 趋势清晰度扣分

- 4H EMA20 与 EMA50 严重缠绕/频繁交叉（>3 次在最近 20 根 K）: -12 分
- 轻微缠绕（刚金叉/死叉后分开<5 根 K）: -5 分
- 已完全分开且斜率健康: 0 分

### 最终五档状态判定

1. HIGH_CONFIDENCE_TREND（≥48 分） → 全力执行 A 流程，仓位 100%
2. TREND_EMERGING（32~47 分 且 日线已转多 或 4H 已金叉） → 允许开仓，仓位 ×0.75，止损距离 ×0.9
3. BOTTOM_REVERSAL_SETUP（见下方独立条件） → 小仓位打剧本 D+
4. CLEAR_RANGING（≤15 分 且 EMA 严重缠绕） → wait
5. LOW_CONFIDENCE_MIXED（其他） → 默认 wait，只允许极高级别剧本 D+

### BOTTOM_REVERSAL_SETUP / TOP_DISTRIBUTION_SETUP 独立触发条件

以下任一组 4 条同时满足即触发（优先级高于普通状态）：

**【多头 - 底部反转】**

1. 日线 MACD 柱子明显收窄或已翻红 ≤5 根 K
2. 价格在重要支撑区域反复测试 ≥2 次
3. 最近 48 小时内出现过放量阳线或锤子线/吞没形态
4. Top_Traders_L/S_Ratio ≤ 0.4 或 Market_L/S_Ratio ≤ 0.6

**【空头 - 顶部派发】**

1. 日线 MACD 柱子明显收窄或已翻绿 ≤5 根 K
2. 价格在重要阻力区域反复测试 ≥2 次（前高、Daily EMA200 上方、整数关口等）
3. 最近 48 小时内出现过放量阴线或射击之星/吞没形态
4. Top_Traders_L/S_Ratio ≥ 3.0 或 Market_L/S_Ratio ≥ 2.5（极端多头情绪）

## A. 趋势开仓流程（适用于 HIGH_CONFIDENCE_TREND & TREND_EMERGING）

TREND_EMERGING 状态自动：仓位 ×0.75，初始止损距离 ×0.9

### 剧本识别

- 【剧本 A】趋势回踩吸筹: Price 在 4H EMA20 ± 0.5×ATR 区间（基础分 80）
- 【剧本 B】盘整突破: 最近 6 根 15min K 线高低点价差 < 0.8×ATR（基础分 75）
- 【剧本 C】趋势延续突破（基础分 70）
- 【剧本 D】底部高置信反转（BOTTOM_REVERSAL_SETUP 触发时使用）
  - 基础分 95
  - 必须同时满足：1H/4H MACD 底背离 + 情绪极端 + 放量关键反转 K 线 + 3min MACD 金叉/死叉确认
  - 风险 0.6%~0.9%，止损 ≤0.6×ATR，持仓上限 8 小时

### 信号增强与过滤

- 3 分钟 MACD 与 15 分钟同向金叉/死叉 → +10 分
- 3 分钟成交量突然放大(>2 倍平均) → +5 分
- 最终信号分 < 75 → wait
- H1_MACD 明确背离 → wait
- RSI 过热过滤：多头趋势 15min RSI(14)>65 或 空头趋势<35 → -10 分
- 3 分钟 RSI(7)>80 或<20 → -5 分

### 仓位计算

- 正常风险 1.8%
- TREND_EMERGING: 1.8% × 0.75
- 剧本 D+/BOTTOM_REVERSAL: 0.6%~0.9%
- 山寨上限 75%, BTC/ETH 85%，再 ×0.92 安全缓冲

## B. 低置信度市场流程（LOW_CONFIDENCE_MIXED）

- 默认：wait
- 唯一例外：剧本 D+信号分 > 92 可小仓位进场

## C. 持仓管理协议

### 止损有效性强制检查

- SHORT: 新止损必须 ≥ 当前价 × 1.002，若计算值更低则用 当前价 × 1.005
- LONG: 新止损必须 ≤ 当前价 × 0.998，若计算值更高则用 当前价 × 0.995

### 黄金法则

- 任何止损调整必须确保风险只减不增
- SHORT: new_stop_loss ≤ current_stop_loss
- LONG: new_stop_loss ≥ current_stop_loss

### 状态驱动行动

1. NO_STOP_LOSS → 立即设置初始止损（SHORT: entry + 1.5×4H_ATR；LONG: entry - 1.5×4H_ATR）
2. STAGE_1（R:R < 0.8） → 默认 hold，仅同时满足 4 条件才谨慎收紧
3. STAGE_2（0.8 ≤ R:R < 1.5）
   - R:R ≥ 1.0 → 移至保本附近（LONG: entry + 0.1× 初始风险；SHORT: entry - 0.1× 初始风险）
4. STAGE_3（R:R ≥ 1.5）
   - R:R ≥ 2.5 → partial_close 40%
   - R:R ≥ 4.0 → partial_close 60%
   - 追踪止损：取 4H_EMA20 与 entry±2.0×ATR 中更优者，只有改善 ≥ 0.25×ATR 且不回吐超 0.5R 才执行

## R:R 计算与状态转换

- 初始风险 = |入场价 - 初始止损价|
- 当前盈利距离 = |当前价 - 入场价|
- R:R = 当前盈利距离 / 初始风险
- R:R < 0.3 → STAGE_1_INITIAL_RISK
- 0.3 ≤ R:R < 0.8 → STAGE_1_RISK_REDUCTION
- 0.8 ≤ R:R < 1.5 → STAGE_2_RISK_REMOVAL
- R:R ≥ 1.5 → STAGE_3_TRAILING

## 极端风险控制

- 剧本 D+连续 3 次亏损后，24 小时内禁止同方向再打反转
- 同一持仓 30 分钟内最多调整 1 次止损
- 市场异常波动(ATR>2 倍平均)时，停止所有止损优化

记住：最好的调整有时是不调整。给交易足够的呼吸空间。
